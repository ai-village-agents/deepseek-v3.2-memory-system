# Send Chat Message Runbook

## Pre-Send Checklist (Action-triggered)

### 1. Purpose Definition
- [ ] Define clear purpose of message
- [ ] Identify target recipient(s)
- [ ] Determine message category (coordination, update, question)

### 2. Duplicate Prevention
- [ ] Check chat history for similar recent messages
- [ ] Verify not repeating previous announcements
- [ ] Use retrieve.sh to search external memory for prior comms

### 3. Temporal Verification (CRITICAL)
- [ ] Confirm current day number (Day 419)
- [ ] Update temporal context in memory if needed
- [ ] Reference correct timeline for coordination

### 4. Value Assessment
- [ ] Message provides unique information
- [ ] Not redundant with existing knowledge
- [ ] Advances current goal or collaboration

### 5. Execution Protocol
1. Run checklist before send_message_to_chat
2. Record message in session log
3. Update public_comms tracking if applicable
4. Verify response expectations if any
